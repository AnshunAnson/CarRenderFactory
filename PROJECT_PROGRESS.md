## 1. 项目阶段总览

- **当前阶段**：M2-M3（核心玩法系统落地与联调）
- **整体完成度（估算）**：**45%**
- **可运行能力**：具备基础角色、近战克制、气势衰减/恢复、通用技能接口、基础排行榜计算能力
- **主要瓶颈**：缺少 UE 运行态联调验证（当前容器无 UBT / 无 Editor）

---

## 2. 已完成内容（代码层）

### 2.1 角色与属性基础
- `AOTA_Character` 已提供基础属性读写接口：`GetQi / GetHealth / GetGold / GetAttackPowerMultiplier / GetKillCount`
- 已实现死亡判定、击杀计数累加、技能冷却状态追踪
- 已实现移动速度倍率应用（基于 `CharacterMovement->MaxWalkSpeed`）

### 2.2 气势系统（Qi）
- `UOTA_QiComponent` 已支持：
  - 运行时 owner 初始化
  - 非摸金状态自动回气
  - 摸金状态每秒掉气
  - Qi 档位切换（High / Normal / Low / Critical）
- Qi 档位影响已接入角色倍率更新（攻击、移速、回气）

### 2.3 战斗与克制
- `UOTA_CombatComponent` 已实现三角克制判定与伤害应用
- 攻击伤害已接入角色攻击倍率（支持成长系统影响战斗）
- 气盾生效时可抵消一次伤害

### 2.4 通用技能（PRD 对齐）
- 已实现三个通用技能入口与资源约束：
  - 气盾（Qi Shield）
  - 瞬步（Dash）
  - 探宝（Treasure Sense）
- 已实现：Qi 消耗、公共冷却、持续时间计时器、关键状态标记
- `PlayerController` 已直连角色技能调用（并保留 GAS tag 路径）

### 2.5 摸金成长与结算
- 金币修改逻辑已实现：
  - 非负限制
  - 50 金币一档成长（上限 3 层）
- 已修复倍率叠加模型：
  - **Qi 态倍率** 与 **金币倍率** 独立存储
  - 通过统一函数汇总，避免互相覆盖
- `GameMode` 已支持基础分数计算与排序接口（击杀 + 金币）

## 4. 风险与建议

### 主要风险
- 缺少真实 Editor/PIE 环境验证，代码正确性与“实际可玩性”之间仍有差距
- 多系统耦合（Qi、Gold、Combat、Skill）后，数值平衡风险上升

### 建议下阶段优先级（P0 -> P2）
- **P0**：在 UE5.7 环境完成编译 + PIE 多人联调 + 基础回归
- **P1**：落地摸金点 Actor（刷新、交互、奖励池）
- **P1**：补齐技能可视化反馈与冷却 UI
- **P2**：开始小规模平衡测试并记录参数变更
