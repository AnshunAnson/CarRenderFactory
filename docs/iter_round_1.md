# Iteration Round 1 设计文档

## 1) 当前系统核心冗余 / 耦合 / 错误风险点
- **迭代流程未固化**：当前仓库缺少标准化的“轮次设计-执行-补丁归档”单一入口，导致每轮执行方式可能分叉。
- **产物命名约束缺失**：没有统一 `iter_round_{N}` 命名规范，后续追溯时容易出现文件命名不一致。
- **变更与设计映射弱**：尚无“设计文档先行”的固定模板，代码/文档改动可能与目标不一致。

## 2) 本轮删减 & 收敛策略
- **删减目标**：删减“临时口头约定流程”，收敛为固定文件化流程。
- **收敛动作**：仅新增一份轮次设计文档，不触碰业务代码与配置，先把迭代闭环的最小约束落盘。
- **边界约束**：本轮不引入新的脚本系统，不扩展新模块，避免过度工程化。

## 3) 保留的最小闭环定义
最小闭环仅包含：
1. 明确本轮问题清单；
2. 明确删减/收敛策略；
3. 明确目标结构；
4. 生成并保存本轮 diff 补丁。

## 4) 明确删除 / 合并 / 收敛的模块 / 文件清单
- **删除**：无（首轮先固化流程，不删除业务资产，避免误伤）。
- **合并**：无。
- **收敛**：将“本轮执行规范”收敛到 `docs/iter_round_1.md` 单文件。

## 5) 迭代后目标结构
- `docs/iter_round_1.md`：本轮设计与执行约束。
- `iter_round_1_diff.patch`：本轮相对上轮（基线）变更补丁。

## 6) 与本轮改动的一致性约束
- 本轮仅允许新增：
  - `docs/iter_round_1.md`
  - `iter_round_1_diff.patch`
- 不允许其它设计外变更。
