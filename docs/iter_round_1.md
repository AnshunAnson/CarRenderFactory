# Iteration Round 1 设计文档

## 1) 当前系统核心冗余 / 耦合 / 错误风险点

1. 迭代流程目前仅存在于自然语言指令中，缺乏可复用的结构化落盘，导致执行口径容易漂移。
2. “上一轮产物 -> 新工作目录 -> 设计 -> 改动 -> patch”闭环没有统一目录约束，后续轮次可追溯性弱。
3. 轮次产物命名规则未固化，容易出现人工命名分叉（round1 / r1 / iter-1）。

## 2) 本轮删减 & 收敛策略

1. **删减隐式流程**：把口头流程收敛到固定目录与固定文件名。
2. **删减命名分叉**：仅保留 `iter_round_{N}` 与 `iter_round_{N}_diff.patch` 两种命名。
3. **删减入口分叉**：本轮只建立一个最小闭环入口（Round 1 固化），不引入额外框架或插件。

## 3) 保留的最小闭环定义

最小闭环仅保留以下五步：
1. 轮次目录初始化
2. 设计文档落盘 (`docs/iter_round_{N}.md`)
3. 按设计做改动
4. 生成 patch (`iter_round_{N}_diff.patch`)
5. 输出轮次总结

## 4) 明确删除 / 合并 / 收敛的模块 / 文件清单

- 删除：无（首轮以流程固化为主，不进行业务代码删改）。
- 合并：将“命名约定 + 产物目录”统一收敛到 `iterations/round_1/`。
- 收敛：新增 `iterations/round_1/README.md` 作为 Round 1 最终产物索引。

## 5) 迭代后目标结构

```text
/docs
  iter_round_1.md
/iterations
  /round_1
    README.md
/iter_round_1_diff.patch
```
