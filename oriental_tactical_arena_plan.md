# 东方战术竞技 (OTA) Demo 实施计划

## 项目概述

**项目名称**: 东方战术竞技 (Oriental Tactical Arena)  
**版本**: v3.0 最终版・深度研究版  
**类型**: 4人自由混战（全敌对）对战 Demo  
**核心参考**: Sifu架势系统 + Absolver武学博弈 + Deadlock移动深度 + 永劫无间三角克制

---

## 一、核心设计理念

### 1.1 气势系统 (Qi Stance System)

基于 Sifu 研究的防御体系四层设计：

```
┌─────────────────────────────────────────────────────────────┐
│                    气势系统 (100点)                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  气势获取:                                                   │
│  • 命中任意对手 → 涨气势                                    │
│  • 格挡任意对手 → 涨气势                                    │
│  • 压制任意对手 → 涨气势                                    │
│                                                             │
│  气势流失:                                                   │
│  • 被任意对手压制 → 掉气势                                  │
│  • 摸金读条期间 → 每秒-10点                                 │
│                                                             │
│  气势阈值效果:                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 80-100: 攻击力+10%，气势恢复+20%                    │   │
│  │ 50-79:  正常状态                                    │   │
│  │ 20-49:  攻击力-10%，移动速度-5%                     │   │
│  │ 0-19:   攻击力-20%，无法使用技能                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 三角克制系统

```
近战三角克制:
    白刀(普通攻击) → 红刀(振刀架势) → 蓝刀(霸体攻击) → 白刀

振刀核心参数 (v2.0):
┌─────────────────────────────────────────────────────────────┐
│  基础振刀窗口: 0.15s (适配P2P多目标同步)                    │
│  高延迟补偿: RTT > 150ms 时自动扩展至 0.3s                  │
│  完美振刀窗口: 0.05s (反弹子弹)                             │
│  振刀成功收益: 造成硬直 + 反击机会                          │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 近战击开子弹

```
多目标偏转机制:
• 一次格挡可以同时偏转所有朝向你的子弹
• 解决多方向火力的防御难题
• 新手也能轻松应对多人攻击
```

### 1.4 摸金博弈机制

```
┌─────────────────────────────────────────────────────────────┐
│                    摸金博弈系统                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  摸金点刷新:                                                 │
│  • 地图危险区域（如地图中心）随机刷新宝箱                   │
│  • 靠近宝箱进行 2 秒读条开启                                │
│                                                             │
│  收益:                                                       │
│  • 10-30 金币                                               │
│  • 小概率掉落稀有临时道具（振刀护盾、远程飞镖）            │
│  • 每攒够 50 金币 → 攻击力+5%，气势恢复+10%（最多3层）     │
│                                                             │
│  风险:                                                       │
│  • 摸金读条期间气势每秒-10点                                │
│  • 无法防御和攻击                                           │
│  • 容易被偷袭                                               │
│                                                             │
│  结算规则:                                                   │
│  • 综合计算「击杀数」与「摸金金币数」                       │
│  • 发育型玩家也有生存空间                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.5 通用技能系统

```
3个极简通用技能（所有玩家共享）:

气盾 (消耗20气势):
• 生成持续2秒的临时护盾
• 抵挡一次任意攻击
• 适合被多人集火时应急保命

瞬步 (消耗15气势):
• 快速向前冲刺3米
• 赶路摸金、追击残血、脱离战场
• 和移动系统气势交换机制适配

探宝 (消耗10气势):
• 显示地图上所有宝箱位置，持续5秒
• 专门适配摸金玩法
• 让发育型玩家快速找到摸金点

公共冷却: 8秒
```

### 2.3 输入缓冲系统

```
┌─────────────────────────────────────────────────────────────┐
│                    输入缓冲设计                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  缓冲窗口: 0.2秒 (12帧)                                     │
│                                                             │
│  缓冲队列结构:                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ struct FInputBufferItem                              │   │
│  │ {                                                    │   │
│  │     EInputAction Action;      // 输入类型           │   │
│  │     float Timestamp;          // 输入时间           │   │
│  │     bool bConsumed;           // 是否已消耗         │   │
│  │ }                                                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  连招输入处理:                                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 1. 玩家在攻击动画期间按下下一个攻击                  │   │
│  │ 2. 输入存入缓冲队列                                  │   │
│  │ 3. 动画到达"可取消帧"时检查缓冲                      │   │
│  │ 4. 如果有有效输入，触发下一个攻击                    │   │
│  │ 5. 否则返回 CombatIdle                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  可取消帧设计:                                               │
│  • 白刀攻击: 持续帧结束后 4帧内可取消                       │
│  • 红刀振刀: 失败后恢复帧全程不可取消                       │
│  • 蓝刀霸体: 全程不可取消                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 六、核心假设验证计划

### 6.1 三个必须验证的核心假设

| 假设 | 验证方法 | 成功标准 |
|------|---------|---------|
| 气势双向博弈比永劫振刀更有深度 | M4单机测试 | 玩家能感知气势差变化带来的博弈 |
| 近战击开子弹在混战中可操作 | M2战斗测试 | 新手能应对多人攻击 |
| 摸金机制不破坏战斗核心 | M3摸金测试 | 玩家不会全程摸金不战斗 |

### 6.2 应急方案

| 风险 | 应急方案 |
|------|---------|
| 气势系统过于复杂 | 简化为单向消耗 |
| 偏转判定困难 | 增大偏转窗口 |
| 摸金破坏平衡 | 增加摸金风险惩罚 |
